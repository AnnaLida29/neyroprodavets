Как это работает

3 этапа диалога — один результат

«Нейро» ведёт клиента по чёткому сценарию — одно сообщение, один вопрос, без давления

01
Этап 1 · Знакомство

Клиент написал — «Нейро» знакомится

Как только клиент пишет на сайте, агент проверяет базу Supabase по IP. Если клиент новый — создаёт карточку и спрашивает имя и нишу. Только один вопрос — и ждёт ответа.

Клиент: "хочу сайт" 🤖 Нейро: Привет! Я рад помочь вам с сайтом 😊 Как я могу к вам обращаться?
02
Этап 2 · Выявление потребности

Узнаёт нишу и ситуацию — обновляет базу

После знакомства «Нейро» спрашивает: есть ли сайт или создаём с нуля? Каждый ответ клиента сохраняется в Supabase — имя, ниша, сайт, история переписки. Агент помнит контекст даже если клиент вернётся через неделю.

Клиент: "Меня зовут Иван, у меня стройка" 🤖 Нейро: Иван, отлично! Стройка — сильная ниша. У вас уже есть сайт или создаём с нуля? 📋 База: client_name="Иван", client_niche="стройка" → Статус: Qualification in Progress
03
Этап 3 · Презентация и контакт

Называет цены, берёт телефон, уведомляет вас

Когда клиент рассказал задачу — «Нейро» кратко называет цены (от 1190 BYN) и просит номер телефона. Как только телефон получен — автоматически отправляет уведомление в Telegram с полной карточкой клиента.

🔥 НОВАЯ ЗАЯВКА С САЙТА! 👤 Имя: Иван Петров 📞 Телефон: +375 XX XXX XX XX 💼 Ниша: Стройка 🌐 Сайт: нет 📝 Итог: Нужен сайт с нуля, звонить после 18:00 → Статус: Qualified Ready for Contact
Возможности

4 режима работы агента

«Нейро» сам определяет, что нужно сделать с каждым сообщением клиента

🆕

create_new_lead

Новый клиент впервые написал на сайте. Агент создаёт карточку в Supabase с IP, первым сообщением и статусом NEW — и начинает знакомство.

✏️

update_lead_details

Клиент уже есть в базе. Агент узнаёт его по IP, обращается по имени и дополняет карточку — ниша, сайт, детали проекта, история диалога.

🔥

qualify_and_notify

Клиент оставил телефон — лид квалифицирован. Статус меняется на «Ready for Contact» и мгновенно летит уведомление в Telegram вашей команде.

💬

general_conversation

Клиент задаёт общие вопросы. Агент отвечает, сохраняет историю переписки и мягко ведёт к следующему этапу квалификации.

🧠

Память разговоров

Postgres Chat Memory хранит последние 10 сообщений в контексте. Клиент может вернуться через неделю — агент продолжит с места остановки.

🎯

Один вопрос за раз

Главное правило агента — никогда не задавать два вопроса в одном сообщении. Только один вопрос, пауза, ответ. Так клиент не перегружается.
